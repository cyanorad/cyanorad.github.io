# Graph View Guide

Using the graph visualization to explore your knowledge network.

## Opening Graph View
- **Keyboard**: Cmd/Ctrl + Shift + G
- **Menu**: View → Graph View

## Features
- **Nodes**: Represent individual notes
- **Edges**: Show links between notes
- **Colors**: Can represent tags or file types
- **Force-Directed**: Physics simulation shows relationships

## Best Practices
- Keep related concepts close in the graph
- Isolated notes may need better linking
- Look for clusters to identify topic areas
- Use local graph (bottom right) for focused view

## Interpreting the Graph
- **Hub notes**: Central ideas with many connections
- **Clusters**: Topic areas with dense connections
- **Bridges**: Notes connecting different domains
- **Orphans**: Disconnected notes needing integration

## Optimizing Your Graph
- Write bidirectional links
- Use consistent naming conventions
- Create MOCs to centralize topics
- Regularly review and add missing links
