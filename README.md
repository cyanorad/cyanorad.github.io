# Learning Vault - Zettelkasten in Obsidian

A comprehensive Obsidian vault setup designed for maintaining learning notes using the zettelkasten methodology.

## 🏗️ Vault Structure

```
vault/
├── 00-Inbox/              # Quick capture area
├── 01-Literature/         # Sources, papers, citations
├── 02-Fleeting-Notes/     # Temporary thoughts
├── 03-Permanent-Notes/    # Your knowledge base
├── 04-Index-and-Maps/     # Navigation & MOCs
├── 05-Logs/               # Daily notes & journals
└── _Templates/            # Reusable templates
```

## ✨ Key Features

### 1. **Zettelkasten System**
- Atomic, self-contained notes
- Bidirectional linking
- Unique identifiers (timestamps)
- Regular refinement and linking

### 2. **Graph Visualization**
- Visual knowledge network
- Identify topic clusters
- Keyboard shortcut: `Cmd/Ctrl + Shift + G`
- See connections between ideas

### 3. **Citation Management**
- BibTeX bibliography system
- Literature notes for sources
- Citekey references in notes
- Track where ideas come from

### 4. **Daily Notes & Logging**
- Daily research journal template
- Calendar integration
- Track learning progress
- Reflective practice

### 5. **Maps of Content (MOCs)**
- Topic-level organization
- Navigation guides
- Concept maps
- Thematic clustering

## 🚀 Getting Started

1. **Clone this vault** into your Obsidian installation
2. **Install recommended plugins**:
   - Dataview (organize and query notes)
   - Citation Plugin (manage bibliography)
   - Obsidian Git (version control)
   - Templater (template automation)
   - Calendar (navigate daily notes)
   - Excalidraw (visual diagrams)

3. **Open the Getting Started guide**: [[MOC-Getting-Started]]

4. **Start capturing**: Use Quick Capture template in [[00-Inbox]]

5. **Process regularly**: Move ideas to [[02-Fleeting-Notes]] and then [[03-Permanent-Notes]]

## 🔧 Core Plugins

- **Graph**: Visualize your knowledge network

## 🏃 Best Practices

1. **Daily Capture**: Spend 10-15 minutes each day capturing ideas
2. **Weekly Review**: Process fleeting notes into permanent notes
3. **Monthly MOC Update**: Refine maps of content
4. **Link Generously**: Use backlinks to build the network
5. **Source Everything**: Document where ideas come from
6. **Use Templates**: Maintain consistency across notes

## 📊 Example Workflow

```
IDEA → Inbox (Quick Capture)
    ↓
    → Fleeting Note (in 1-2 days)
    ↓
    → Permanent Note (atomic, linked)
    ↓
    → Add to MOC (organize by topic)
    ↓
    → Graph View (visualize connections)
```
